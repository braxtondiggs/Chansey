The Five Stages

01

Optimize

Grid-search parameter space with walk-forward analysis to find the best config.
Walk-forward train → test → step L1: 75 combos L5: 30 combos queue · optimization
Produces
  • OptimizationRun
  • bestParameters (JSON)
  • bestScore & improvement %
🔒 improvement ≥ 3% & score ≥ threshold
02

Historical Backtest

Run the optimized parameters over a held-out historical window to test real-world performance.
Window 45 – 90 days Capital $10,000 Type HISTORICAL queue · backtest-historical
Produces
  • Backtest (trades, snapshots)
  • Sharpe · drawdown · win rate
  • Total return %
🔒 ≥ 1 trade executed
03

Live Replay

Replay against the most recent ~30 days of real market data — the ultimate out-of-sample test.
Window ~30 days Pacing adjustable Pause/Resume supported queue · backtest-replay
Produces
  • Backtest (LIVE_REPLAY)
  • Degradation vs historical
  • Composite pipeline score
🔒 composite score ≥ 30 / 100
04

Paper Trade

Trade with simulated capital against live market ticks — verify signal quality in real time.
L1: ≥ 50 trades L5: ≥ 30 trades Max 30 days queue · paper-trading
Produces
  • PaperTradingSession
  • Live signal P&L
  • Exit behavior (SL / TP / trail)
🔒 Sharpe ≥ 0.3  ·  DD ≤ 45%  ·  return ≥ 0
05

Completed

Synthesize all stage results into a single recommendation — ready for promotion evaluation.
pipelineScore 0 – 100 Recommendation generated Report archived
Produces
  • PipelineSummaryReport
  • DEPLOY / REVIEW / REJECT
  • Audit log entry

Pipeline Recommendations

◆ Deploy

Ready for production

Strategy passes all quality gates — automatically eligible for shadow → live promotion.

score ≥ 70 · Sharpe ≥ 1.0 · DD ≤ 25%
◆ Needs Review

Manual evaluation

Moderate performance — requires human judgment on whether to promote or iterate.

score 30 – 69 · Sharpe ≥ 0.5 · DD ≤ 40%
◆ Do Not Deploy

Insufficient quality

Strategy fails one or more critical thresholds — returned for parameter revision.

score < 30 · or thresholds failed
